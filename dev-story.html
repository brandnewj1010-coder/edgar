Background

재무 공시, 보고 싶었는데
보기가 너무 어려웠습니다

재무제표를 직접 읽어보려고 DART에 들어가보면 수십 개의 탭, 방대한 XML 파일, 회사마다 다른 항목 이름… 정보는 분명히 공개돼 있는데 정작 쓸 수 있는 형태로 꺼내기가 쉽지 않았습니다. "기업명 하나 치면 핵심 수치가 바로 나오면 안 될까"라는 단순한 질문에서 시작했습니다.

Data Source

공식 정부 API를
직접 연결했습니다

공시런의 국내 데이터는 네이버·증권사 화면에서 긁어온 것이 아니라, 금융감독원이 운영하는 DART(전자공시시스템) 공식 API에서 직접 가져옵니다. 상장사가 금감원에 제출하는 재무제표 원문 그대로입니다.

📋 DART OpenAPI — opendart.fss.or.kr

금융감독원이 제공하는 전자공시 원문 API

국내 전 상장사가 사업보고서·분기보고서에 제출하는 재무제표를 API로 직접 조회할 수 있습니다. 증권사 화면에 표시되는 수치도 결국 이 데이터를 기반으로 합니다. 미국 기업은 SEC EDGAR의 XBRL 공시 데이터를 동일한 방식으로 연결했습니다.

Behind the Scenes

API가 열려 있다고
바로 쓸 수 있는 건 아니었습니다

공식 API를 실제 서비스에 연결하는 과정에서 마주친 것들입니다.

🔑

공시 시스템은 우리가 아는 종목코드를 쓰지 않습니다

삼성전자의 KRX 종목코드는 005930이지만, DART API는 금감원이 별도로 부여한 고유번호를 요구합니다. 두 코드는 전혀 다릅니다. 기업마다 이 번호를 직접 찾아서 매핑해야 했고, 처음에 입력한 20개 이상의 코드 중 절반 가까이 틀려서 하나하나 다시 검증했습니다.

전 상장사 코드를 자동으로 불러오는 방식으로 해결
📄

같은 항목인데 회사마다 이름이 다릅니다

'매출액'이라는 항목을 어떤 회사는 "매출액", 다른 회사는 "수익(매출액)", 또 다른 회사는 "영업수익"이라고 씁니다. 단순히 이름으로 찾으면 데이터가 빠지거나 엉뚱한 값이 나옵니다. IFRS 국제 표준 코드로 항목을 식별하도록 파싱 방식을 전면 재설계했습니다.

국제 표준(IFRS) 코드 기반 추출로 어느 기업이든 일관되게
⚡

전체 상장사 목록을 매번 받아오면 10초가 걸렸습니다

DART가 제공하는 상장사 목록 파일은 압축 해제 시 수십만 줄에 달합니다. 검색할 때마다 이걸 다운로드하면 응답이 너무 느렸습니다. 서비스 빌드 시점에 파일을 미리 가공해서 CDN에 올려두는 방식으로 바꿨더니 응답이 0.5초로 줄었습니다.
